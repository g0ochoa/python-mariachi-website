# Sprint Summaries

This directory contains daily summaries from each sprint, documenting what was accomplished during team meetings and work sessions.

## Naming Convention

Files follow the pattern: `sprint_{number}_day_{day}.md`

Examples:
- `sprint_2_day_1.md`
- `sprint_2_day_2.md`
- `sprint_3_day_1.md`

## Standard Format

Each sprint summary should include:

1. **Date & Participants**: When the meeting/work session occurred and who was involved
2. **Objectives**: What we planned to accomplish
3. **What We Did**: Detailed breakdown of work completed
4. **Decisions Made**: Key technical or project decisions
5. **Blockers/Issues**: Any problems encountered
6. **Next Steps**: What needs to happen next
7. **Action Items**: Specific tasks assigned to team members

## Purpose

These summaries serve as:
- Project documentation and history
- Reference for team members who missed sessions
- Context for future work
- Evidence of progress for stakeholders
